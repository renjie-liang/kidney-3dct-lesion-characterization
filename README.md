# Multi-Granularity 3D Kidney Lesion Characterization from CT Volumes

Code for the paper *Multi-Granularity 3D Kidney Lesion Characterization from CT
Volumes*. We reformulate kidney-CT characterization as a **per-lesion
set-prediction** task: from one 3D kidney CT volume, a single model predicts a
variable number of lesions per kidney, each with four clinical attributes (type,
size, enhancement, attenuation), while also serving coarser side-level outputs.

The core model is **LesionDETR**, a DETR-style head with size-distance Hungarian
matching and a hierarchical loss that aggregates per-slot predictions into
side-level objectives across three label granularities:

- **L1** — side-level abnormality (per kidney)
- **L2** — side-level lesion type (cyst / solid) and max size
- **L3** — per-lesion attributes

> **Results:** see [`RESULTS.md`](RESULTS.md) for the paper's numbers
> (Tables 2–4). They require the private UF Health dataset and are **not**
> reproducible from the public/synthetic data here.

## Data availability

The UF Health dataset (real patient CT + report-derived labels) **cannot be
released** due to patient-privacy restrictions. To keep the code runnable, this
repository works with two small public datasets, hosted separately on
Hugging Face: **[kidney-3dct data](https://huggingface.co/datasets/LiangRenjie/kidney-3dct-lesion-characterization)**.

| Dataset | Purpose | Granularity | License |
|---|---|---|---|
| Synthetic (generated by this repo) | run the full pipeline, incl. per-lesion L3 | L1/L2/L3 | MIT |
| KiTS23 subset (6 real cases) | external-validation code path | L1/L2 | CC BY-NC-SA 4.0 |

No PHI, radiology reports, patient identifiers, or private annotations are
included anywhere in this repository. See [`DATA.md`](DATA.md).

## Installation

```bash
git clone https://github.com/renjie-liang/kidney-3dct-lesion-characterization
cd kidney-3dct-lesion-characterization
python -m venv .venv && source .venv/bin/activate   # or conda/uv
pip install -r requirements.txt
```

Python ≥ 3.10. A CUDA GPU is recommended for training; the smoke test also runs
(slowly) on CPU.

## Quick start (no download required)

Generate a tiny synthetic dataset and train LesionDETR end-to-end on it:

```bash
python tools/generate_synthetic_data.py --out datasets   # writes fake cases under datasets/UF_Kidney_CT
python train.py --config-name smoke                       # 2 epochs, from-scratch encoder, tiny volumes
```

This verifies the whole pipeline (data loading → encoder → LesionDETR →
hierarchical loss → L1/L2/L3 evaluation). It is a smoke test, **not** a paper
result.

## Configuration & paths

Paths are resolved from environment variables (with runnable defaults), so no
machine-specific paths are baked in:

| Variable | Meaning | Default |
|---|---|---|
| `KIDNEY_DATA_ROOT` | UF-format dataset (`final_dataset/`) | `datasets/UF_Kidney_CT/final_dataset` |
| `KITS23_ROOT` | processed KiTS23 (`step4_cropped/`, `labels.json`) | `datasets/KiTS23/processed` |
| `WEIGHTS_ROOT` | pretrained encoder weights | `weights` |
| `KIDNEY3DCT_ROOT` | output root for `experiments/` | repo root |

Training is configured with [Hydra](https://hydra.cc); `configs/default.yaml`
holds the defaults and any field can be overridden on the CLI, e.g.
`python train.py encoder=suprem ct_level=L3 label_level=L3_D lr=5e-4`.

## Pretrained encoders

The paper uses three pretrained 3D encoders. Download them from their official
repositories and place the weights under `WEIGHTS_ROOT`:

| Encoder | File (expected under `WEIGHTS_ROOT/`) | Source |
|---|---|---|
| SuPreM (default) | `SuPreM/supervised_suprem_swinunetr_2100.pth` | <https://github.com/MrGiovanni/SuPreM> |
| VoCo | `VoCo/VoComni_B.pt` | <https://github.com/Luffy03/VoCo> |
| CT-CLIP (CTViT) | `CT-CLIP/CT-CLIP_v2.pt` (or set `CT_CLIP_WEIGHT`) | <https://github.com/ibrahimethemhamamci/CT-CLIP> |

`encoder=from_scratch` requires no weights and is used by the smoke test.

## Repository layout

```
train.py                 # main training (L1 / L2 / L3) — Hydra entry point
eval_kits23.py           # zero-shot external validation on KiTS23
eval_l3_full.py          # detailed per-lesion (L3) evaluation
finetune_kits23.py       # optional KiTS23 fine-tuning
configs/                 # default.yaml + smoke.yaml
models/                  # SwinUNETR / CT-ViT encoders, classifier, LesionDETR (l3_heads)
data/                    # KidneyCTDataset, KiTS23 loader, few-shot sampler
analysis/                # per-lesion AP metric, bootstrap CI, probing, AP validation
detection_training/      # self-contained per-lesion head comparison (Table 3)
tools/
├── generate_synthetic_data.py   # synthetic dataset generator (data contract spec)
├── prepare_kits23.py            # reproduce KiTS23 in our format from the official download
└── process_kits23/              # the 5-step KiTS23 preprocessing pipeline
```

## Reproducing the experiments (with the real dataset)

With a `KIDNEY_DATA_ROOT` in the expected format, the paper's tables map to:

| Paper | Command |
|---|---|
| Table 2 (input × granularity) | `python train.py encoder=suprem ct_level={L1,L2,L3} label_level={L1,L2,L3_D} mask_strategy={none,binary,7class}` |
| Table 3 (per-lesion heads) | `bash detection_training/scripts/submit_all.sh` then `python detection_training/scripts/aggregate_results.py` |
| Table 4 (encoders) | `python train.py encoder={suprem,voco,from_scratch} ct_level=L3 label_level=L3_D mask_strategy=binary` |
| External validation | `python tools/prepare_kits23.py --raw <kits23> && KITS23_ROOT=datasets/KiTS23/processed python eval_kits23.py --checkpoint <best_model.pt>` |
| Per-lesion AP validation | `python analysis/test_ap_synthetic.py` |

Edit the cluster placeholders in `detection_training/scripts/submit_all.sh`
(account, partition, environment) before submitting.
